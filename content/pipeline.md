# Content Pipeline

Every piece of content moves through these stages. Track status in the relevant file.

## Stages

```
IDEA → OUTLINE → DRAFT → REVIEW → PUBLISHED → PROMOTED → ARCHIVED
```

| Stage | Definition | Time cap |
|-------|-----------|---------|
| IDEA | Captured in ideas.md — a title or one-line premise | 5 min |
| OUTLINE | Structure written — sections or bullet points | 20 min |
| DRAFT | Full first draft complete | 1–3 hrs |
| REVIEW | Self-reviewed, code tested, facts checked | 30 min |
| PUBLISHED | Live on the platform | — |
| PROMOTED | Shared on LinkedIn + other channels | 15 min |
| ARCHIVED | Done; logged in platform-tracker.md with link | 5 min |

---

## Weekly content workflow (30–60 min/day)

### Monday
- Review ideas.md → pick one for the week
- Write OUTLINE

### Tuesday–Wednesday
- Write DRAFT (don't edit while writing)

### Thursday
- REVIEW + edit
- Prepare accompanying GitHub snippet or diagram

### Friday
- PUBLISH
- Write LinkedIn post announcing it
- PROMOTE across channels

### Ongoing (daily, 10 min)
- Write 1 LinkedIn post (can be short: 150–300 words)
- Comment on 3 posts in target company/person feeds
- Log anything new to learning-log.md

---

## Content from learning (the efficient path)

You shouldn't generate content ideas from scratch.
Content should flow naturally from your learning log:

1. You learn something in `second-brain/learning-log.md`
2. If it has a bridge insight → stub a Medium article in `content/ideas.md`
3. If it has a buildable artifact → stub a project in `github-projects/ideas.md`
4. When artifact is built → write a LinkedIn post (2 hr total)
5. When LinkedIn post gets traction → expand to Medium article (3 hr total)
6. If the topic is visual/complex → record a YouTube walkthrough (4 hr total)

## Content quality checklist

Before publishing anything:

**LinkedIn post**
- [ ] First line is a hook (not "I just published...")
- [ ] Under 1,500 characters (or broken into digestible paragraphs)
- [ ] Ends with a question or call to action
- [ ] One relevant hashtag cluster (3–5 hashtags)

**Medium article**
- [ ] Title contains a search-intent keyword
- [ ] Subtitle explains the value in 1 sentence
- [ ] First paragraph hooks without explanation ("here's why X matters")
- [ ] Code is in code blocks and tested
- [ ] GitHub repo linked if relevant
- [ ] Add to publications: Towards Data Science / Level Up Coding / Better Programming

**YouTube video**
- [ ] Title is searchable (not clever — clear)
- [ ] Thumbnail text is legible at small size
- [ ] First 30 seconds deliver on the title promise
- [ ] Code in description + GitHub link
- [ ] Pinned comment with chapter timestamps

**GitHub repo**
- [ ] README.md has: what it does, why it matters, how to run it, screenshot/demo
- [ ] ATS keywords embedded naturally in README
- [ ] License file present (MIT unless otherwise)
- [ ] At least one passing test or demo script

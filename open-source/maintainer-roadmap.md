# Maintainer Roadmap

Path from first contribution to maintainer status, per project.
This is a 6–18 month journey depending on project and focus level.

---

## Generic path

```
Month 1–2: EXPLORER
  - Study codebase, read all open issues
  - Fix 1–2 documentation issues (easy merges, builds relationship)
  - Introduce yourself in Slack/Discord/Discussions

Month 2–4: CONTRIBUTOR
  - 3–5 merged PRs (mix of docs, bug fixes, small features)
  - Become known by 1–2 core maintainers
  - Start reviewing other PRs (adds value before having write access)

Month 4–6: REGULAR CONTRIBUTOR
  - 10+ total merged contributions
  - Reviewers trust your judgment and suggest you review PRs
  - Write a blog post or video about the project → share with maintainers

Month 6–12: COMMITTER (write access)
  - Maintainer explicitly invites you OR you request it with a track record
  - Merge your own PRs after approval
  - Triage new issues

Month 12–18: MAINTAINER
  - Triage all new issues in your domain area
  - Review and approve PRs from others
  - Participate in roadmap discussions
  - Represent the project publicly (talks, blog posts)
```

# Content Engine

Evidence is only as good as its reach. A GitHub repo nobody finds is invisible.
Content turns artifacts into reputation, and reputation into inbound recruiter calls.

## The flywheel

```
Gap analysis identifies missing skill
        ↓
Learn it (second-brain/learning-log.md)
        ↓
Build a project proving it (github-projects/)
        ↓
Write about it (content/medium.md or content/linkedin-posts.md)
        ↓
Record a walkthrough if complex (content/youtube.md)
        ↓
Log the artifact (evidence/platform-tracker.md)
        ↓
Update profile/my-profile.yaml with evidence link
        ↓
LinkedIn post drives profile views → recruiter inbound
```

## Content types and their ROI

| Type | Time | Reach | SEO | Credibility | Best for |
|------|------|-------|-----|-------------|---------|
| GitHub repo | 1 day–1 week | Medium | Medium | High | Proving technical depth |
| Medium article | 2–4 hrs | High | High | High | Domain knowledge + thought leadership |
| LinkedIn post | 20–30 min | Very High | Low | Medium | Reach + algorithm amplification |
| LinkedIn article | 1–2 hrs | High | Medium | High | Detailed analysis |
| YouTube video | 4–8 hrs | High | High | Very High | Tutorials, walkthroughs, career story |
| Open source PR | 1 hr–1 week | Low-Med | Low | Very High | Peer credibility |
| Conference talk | 4–8 hrs prep | Low | Low | Very High | Authority signal |

## Content pillars (your 4 angles)

Every piece of content should belong to one of these:

1. **The Bridge** — cross-domain insights ("what RTL taught me about distributed systems")
   Audience: engineers, hiring managers who want unusual thinkers
   
2. **The Tutorial** — build something, show every step
   Audience: practitioners learning; recruiters verifying depth
   
3. **The Landscape** — state of a domain you know well
   Audience: non-specialists wanting signal; recruiters researching domain
   
4. **The Career Story** — how you got from X to Y, what you learned
   Audience: people on similar paths; recruiters assessing trajectory

## Cadence targets

| Channel | Frequency | Type |
|---------|-----------|------|
| LinkedIn posts | 2–3×/week | Short (300–500 words) + image/code snippet |
| Medium | 2×/month | Long-form (1,500–3,000 words) |
| GitHub | 1 project/month | Working code + README |
| YouTube | 1×/month | Tutorial or walkthrough (15–30 min) |
| Open source PRs | 2–4×/month | Bug fixes, docs, features |

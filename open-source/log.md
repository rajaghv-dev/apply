## Type key
- **PR** — pull request (code, docs, tests)
- **Issue** — filed a bug or feature request
- **Review** — reviewed someone else's PR
- **Comment** — substantive issue comment (reproduction, diagnosis)
- **Discussion** — participated in design discussion

## Status key
- OPEN — submitted, not yet reviewed
- REVIEWED — feedback received, addressing
- APPROVED — approved, waiting for merge
- MERGED — done ✓
- CLOSED — not accepted (learn from the feedback)

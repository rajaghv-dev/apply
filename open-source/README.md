# Open Source Strategy

## Why open source matters for credibility

A merged PR in a well-known project is worth more than a personal repo.
It proves: you write code others accept, you understand an existing codebase,
you can collaborate with strangers under scrutiny.

The goal hierarchy:
1. **Contributor** — 1+ merged PRs (proves you can navigate unfamiliar code)
2. **Regular contributor** — 5+ merged PRs in the same project (proves persistence + judgment)
3. **Committer** — write access, reviewing others' PRs (proves community trust)
4. **Maintainer** — triaging issues, roadmap input (proves domain authority)

Timeline: 3–6 months per project if focused (2–4 PRs/month).

## Portfolio strategy

Don't spread thin. Pick 2–3 projects and go deep.
Choose projects that:
- Align with your target role domain
- Are actively maintained (PRs reviewed within 2 weeks)
- Have a "good first issue" or "help wanted" label system
- Have audience (stars, users, corporate backing)
